如何搭建个人博客

发表时间：2022-08-05 阅读： 评论：

经验方法 前端 博客

本文提供的博客搭建方法基于Github Pages与Hexo，无需服务器、域名注册、无后端，适合免费白嫖党使用。

准备材料

  • 一台windows电脑
  • 一个github账号
  • 一个前端代码编辑器（WebStorm）（https://www.jetbrains.com/webstorm/download/#section=windows）
  • 安装Node.js
  • 安装git

搭建流程

Github仓库搭建

在Github上注册账号后，创建一个新的仓库。注意该仓库名称应为“你的github用户名.github.io”（username.github.io)。
根据Github Pages的说明介绍,此时github的服务器应该已经为你自动生成了一个网址（https://username.github.io/).
不过目前该网址内没有任何内容，所以直接打开的话会显示404.

如果想要使用别的博客搭建方法或自己编写博客网页的话，可以参照Github Pages中的说明，将仓库拉取到本地，然后自由搭建。 但本文所使用的Hexo搭建方法只要求做完上述一步就ok了。

安装及部署Hexo

在你的电脑上随便找一个地方大一点的地方，新建一个文件夹命个名，作为博客本地存储位置。例如本人的博客选址选在F://MyBlog.
打开WebStorm，在下方的终端框内，定位到该地址。cd F://Myblog

然后使用Node.js的包管理器npm安装Hexo。输入npm install hexo-cli -g开始安装。装好后，在终端输入hexo -v，若出现一堆版本号，则安装成功。

接着输入hexo init，创建博客文件夹。输入前一定要保证当前地址所在文件夹为空，否则会报错。创建完成后，当前文件夹内会出现node_modules/source/themes等许多文件夹。各文件夹含义及作用可见https://hexo.io/zh-cn/docs/setup。

接着输入npm install，安装相关组件；输入hexo server，启动博客服务。点击弹出的链接(http://localhost:4000/ )，如一切正常，则应显示以下页面，表明博客创建成功！

将页面推送到Github Pages

设置Git的user name、和email和SSH

在博客所在文件夹上右击，点击Git Bash Here。在弹出的命令行窗口中依次输入git config --global user.name "yourname"和git config --global user.email "youremail"。
其中的yourname和youremail是你再github上的用户名和注册邮箱。

接着输入ssh-keygen -t rsa，连续回车三下（直到所有对话均确认跑完）。然后在默认路径C:\Users\Administrator.ssh下即可发现两个新生成的文件id_rsa和id_rsa.pub。

登录Github，点击头像下的settings，点击左侧SSH and GPG keys一栏。在右侧输入SSH的名字（随意），然后把id_rsa.pub文件里的内容复制到下面。点击Add SSH key即可。

推送博客项目

打开博客文件夹内的_config.yml文件（该文件为博客的配置文件，大多数设置均位于此文件）。设置该文件最下方deploy内的repo后的地址为你在github内刚刚新建的仓库的地址。

然后安装在git上部署文章的扩展npm install hexo-deployer-git --save。

在终端框内输入hexo g生成静态文件；再输入hexo d即可部署到Github Pages。

此时再打开浏览器，输入网址https://username.github.io/ ，即可查看到你的个人博客界面啦！

博客文件的新建与推送

你的博客文件在本地均会存储于./source/_posts文件夹内。在终端定位到博客所在文件夹，输入hexo new "Filename"即可创建新博客。创建完成后，可发现_posts文件夹内多出一个.md文件。

通过上述方式创建的博客在浏览器端直接可见。如果想创建草稿，可输入hexo new draft "Filename"，之后再将其发布。此外还有多种创建方式，详情可见https://hexo.io/zh-cn/docs/writing.html

所有的博客都将以MarkDown格式书写，可在编辑器内直接编写博客。MarkDown的具体书写方式可参考https://www.jianshu.com/p/806d856cbd4e。

另外，Hexo还支持在MarkDown内对文章分类、添加标签等。详情可见https://hexo.io/zh-cn/docs/front-matter

经过尝试，若想在博客的MarkDown中插入图片，图片链接只能使用网络链接。本地的链接在被推送到Github Pages之后会地址丢失。本人将图片存储在Github仓库内，使用时调取图片网络地址。最后提一句，仓库文件夹命名最好用英文，用中文会像下面一样很难受：

博客编写好后，可运行hexo server先在本地预览。确认无误后，hexo g+hexo d将其推送到Github Pages上。

博客主题的更换

Hexo官网上有许多美观的主题（https://hexo.io/themes/ ），不同的主题应用方式不大一样，具体应用方式应点开对应主题，查看其作者书写的说明文档。

最后

当然，由于本方法搭建的博客使用的服务为Github Pages，其服务器在国外，故在不能科学上网的情况下访问速度较慢。后续可能会尝试解决这个问题。
